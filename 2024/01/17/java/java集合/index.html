Collection

存储null值

  • Hashtable 不允许 null 键或值。
  • TreeMap 和 LinkedHashMap 不允许 null 键

常用方法

  • add()
  • clear()
  • remove()
  • contains()
  • isEmpty()
  • size()

contains()：依赖于equals()方法(没有重写)，比较的是地址

查询多：ArrayList
增删多：LinkedList
去重：HashSet
去重有序：LinkedHashSet
排序：TreeSet

Collections集合工具类
addAll()
shuffle()

遍历方式

迭代器：

  • 遍历后指针不会复位
  • 不能用集合的方法进行增加或删除(可以用迭代器的删除方法remove())

增强for：
所有的单列集合和数组才能用增强for遍历,不会改变集合中原本的数据

Lambda：
arr.forEach(System.out::println);

list

有序，可重复，有索引

方法：
add(int index,E element)
remove(int index)
set(int index,E element)
get(int index)

remove(1) 是删除索引
方法出现重载，优先调用实参和形参一致的方法 这里会调用remove(int index)

列表迭代器
ListIterator iterator = list.listIterator();
可以添加元素

LinkedList

双链表，首尾操作很快

方法
addFirst()
addLast()
getFirst()
getLast()
removeFirst()
removeLast()

set

  • HashSet 无序，不重复，无索引
  • LinkedHashSet 有序
  • TreeSet 可排序

底层是红黑树，不是哈希表

排序规则：

  • 实现Comparable接口重写compareTo
  • 自定义comparator（优先

方法：
put()
remove()
clear()

size()

put:键存在会覆盖，返回被覆盖的值

遍历：

  • 键找值
  • 键值对

HashMap

哈希值
hash = key.hashCode()) ^ (h >>> 16)

对应哈希数组下标：
(n - 1) & hash = hash % n

TreeMap

红黑树

自平衡的二叉搜索树

增删查改性能都很好

  • 节点为红色或黑色
  • 根节点为黑色
  • 没有子或父节点的节点指针属性为nil，视为叶节点，每个叶子节点(NIL)为黑色
  • 对每个节点，从该节点到其所有后代叶节点的简单路径上，均包含相同数目的黑色节点
  • 两个红色节点不能相连
  • 默认添加红色节点

HashMap:效率高
LinkedHashMap:有序
TreeMap:排序
