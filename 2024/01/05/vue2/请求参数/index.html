处理请求参数

发表于2024-01-05|更新于2024-05-21|前端
|阅读量:

处理函数参数

params 是一个对象，其中包含要发送到服务器的查询参数（query parameters）

模板字符串：增强版的字符串，用反引号（`）标识。它可以当作普通字符串使用，也可以用来定义多行字符串，或者在字符串中嵌入变量。

请求参数格式

查询参数

queryString查询参数
查询参数通常用于 GET 请求，它们被附加到 URL 的末尾，并用 ? 开始，参数之间用 & 分隔

请求体

application/x-www-form-urlencoded
这是一种常见的格式，通常用于在POST请求中发送表单数据。参数以键值对的形式出现，并使用&符号进行分隔。键值对中的键和值都会进行URL编码。multipart/form-data可以上传文件（可以同时包含文本字段和二进制数据）

application/json
这是最常见的请求体格式，消息主体是序列化的json
在Vue 3中，如果你传递一个对象作为请求体，axios会默认将这个对象转换为JSON格式的数据，然后设置请求头Content-Type为application/json。

发送请求以及接收

post

  • x-www-form-urlencoded表单数据：可以使用URLSearchParams对象拼接
  • application/json：直接传

接收表单：

接收json

get

URLSearchParams对象
这个对象通常用于发送FormData格式的HTTP请求，特别是当需要发送POST或者PUT等请求且内容编码类型为application/x-www-form-urlencoded或multipart/form-data时。（效果和params一样）

Spring MVC 后端接收参数总结

查询参数：使用@RequestParam注解。
表单数据：使用@ModelAttribute注解（对应整个表单对象）或@RequestParam注解（对应单个表单字段）。
JSON 数据：使用@RequestBody注解，并将请求体中的数据自动绑定到对应的Java对象上。

请求参数实例

application/json：
{ "title": "陕西旅游攻略", "state": "草稿", "categoryId": 2 }

x-www-form-urlencoded表单数据：
username=zhangsan&password=123456

现代的后端框架通常具有对不同请求体格式的解析能力，能够灵活处理 JSON 格式、表单形式等不同的请求体。这种灵活性使得前端可以选择更适合项目需求的传参方式。
