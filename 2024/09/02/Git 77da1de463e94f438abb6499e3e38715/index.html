Git

Created: July 31, 2024 3:17 PM
Tags: tool

区域

  • 工作区
  • 暂存区（缓存）
  • 本地仓库
  • 远程仓库

命令

git add 添加文件到暂存区
git status 查看仓库当前的状态，显示有变更的文件。
git commit 提交暂存区到本地仓库。
git reset 撤销暂存区，回退版本，git reset HEAD
git switch 更清晰地切换分支。
git restore 恢复或撤销文件的更改。
git log 查看提交历史
git checkout 切换分支，没有则创建
git switch dev 切换分支
git checkout 撤销工作区
git reset 撤销暂存区
git revert 创建一个新的提交以撤销指定提交，不改变提交历史。
git branch -d 删除分支
git checkout -b dev 创建分支并切换
git branch -m 分支名 新的分支名 修改分支名字
git remote show origin 查看远程分支
git fetch 拉取远程分支,不合并
git pull 拉取远程分支,合并
git rebase 合并，创建新提交纪录
git remote 远程仓库操作
git push 上传远程代码并合并

撤销

版本回退

git reset --hard HEAD

  • --soft 最安全，因为它只改变 HEAD 的位置；

暂存区和工作区不丢失，下一步commit

  • --mixed 更进一步，会清除暂存区中的更改；

工作区不丢失暂存区丢失，下一步add

  • --hard 则是最激进的，会完全恢复到指定提交的状态。

HEAD^ 回到上一个版本

HEAD~3 往前3个版本

恢复（取消回退）：

git reflog 查看到commit id, reset到这个版本

撤销修改

git checkout -- <file> 丢失工作区修改

git reset HEAD <file> 丢失暂存区的修改

协作

抓取分支

clone远程库只能看到本地master分支

需要创建远程origin的dev分支： git checkout -b dev origin/dev
