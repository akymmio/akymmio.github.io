PXE

Created: July 29, 2024 3:55 PM
Tags: tool

PXE（Preboot Execution Environment，预启动执行环境）是一种允许计算机通过网络启动操作系统而无需本地存储设备的技术。

PXE 是一种基于网络的启动技术，最初由 Intel 开发，旨在提供一种在没有本地存储设备的情况下通过网络启动操作系统的标准。PXE 集成在计算机的 BIOS 或 UEFI 中，允许计算机从网络服务器下载并启动操作系统或其他软件。

原理

pxe服务器

PXE 服务器是提供引导文件和操作系统映像的服务器

  • DHCP服务器 （动态主机配置协议
  • TFTP服务器（简单文件传输协议

pxe客户端

PXE 客户端是通过网络引导的计算机，它需要支持 PXE 协议的网卡或固件（通常在 BIOS/UEFI 中启用）

流程

  1. 启动并请求 IP 地址：

    客户端通过网络启动，并向 DHCP 服务器请求 IP 地址。

  2. 获取引导文件位置：

    DHCP 服务器响应请求，并提供 TFTP 服务器的地址和引导文件名。

  3. 下载引导文件：

    客户端使用 TFTP 从服务器下载引导文件（如 pxelinux.0）。

  4. 执行引导文件：

    客户端执行下载的引导文件，并根据配置加载内核和初始 RAM 磁盘。

  5. 启动操作系统安装：

    客户端根据引导菜单选择合适的引导选项，并从网络服务器加载操作系统安装文件或映像。

文件结构：

  • 确保 TFTP 服务器根目录 /tftpboot 下的文件结构与 dhcpd.conf 中指定的路径一致。

通过这种方式，DHCP 服务器会指示客户端从 TFTP 服务器的 /tftpboot 目录中获取grubx64.efi 和 common/grub.cfg 文件

总结：

  • DHCP 服务器的“根目录”实际上是指 TFTP 服务器的根目录。
  • 在 dhcpd.conf 中指定文件路径时，使用相对于 TFTP 服务器根目录的路径。
  • 确保 TFTP 服务器根目录下的文件结构与 dhcpd.conf 中指定的路径一致。
