在数据库设计中，当一个表中的字段引用了另一个表的主键，但没有在数据库层面上明确声明为外键约束（即物理外键），这样的字段被称为逻辑外键。尽管没有数据库层面的约束保证数据一致性，但在逻辑上和应用程序设计中，这个字段仍然起到了关联两个表的作用。

事务

事务是指作为一个逻辑单元执行的一系列操作，这些操作要么全部成功执行，要么全部回滚到最初状态，以保持数据库的一致性和完整性。

  • 开启事务 start tansaction; begin;
  • 提交事务 commit
  • 回滚事务 rollback

四大特性ACID

  • 原子性（Atomicity）：事务是一个不可分割的操作单元，要么全部执行成功，要么全部回滚到最初状态。如果事务中的任何一个操作失败，整个事务都将被回滚，不会对数据库产生任何永久性的改变。
  • 一致性（Consistency）：事务在执行之前和之后，数据库的状态必须保持一致。这意味着事务必须遵循所有预定义的规则和约束，以确保数据库的完整性。
  • 隔离性（Isolation）：每个事务的执行都应该与其他事务相互隔离，以防止相互干扰。事务应该像独立运行一样，即使在多个事务同时执行的情况下也不会相互干扰。
  • 持久性（Durability）：一旦事务被提交，对数据库所做的更改应该是永久性的，并且应该能够在系统故障或崩溃后恢复。数据库系统应该能够将已提交的事务的更改保存在永久存储器中，以确保数据的持久性。

索引

帮助数据库高效获取数据的数据结构

Mysql默认B+Tree组织结构的索引

语法

  • 创建索引： create [unique]index 索引名 on 表名（字段名…）
  • 查看索引： show index from 表名
  • 删除索引： drop 索引名 on 表名

(主键自动创建主键索引，唯一字段添加唯一索引)

MVCC

多版本并发控制，实现对数据库的并发访问，解决读写冲突的无锁并发控制
