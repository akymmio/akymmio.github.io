请求响应

springboot中的请求处理

  • 简单参数： 定义方法形参，请求参数名和形参变量名相同，如果不一致@RequestParam可以映射
  • 实体参数：请求参数名和形参对象属性名相同，直接定义POJO接受
  • 数组:请求参数名和形参中数组变量名相同，直接使用数组封装
  • 集合：请求参数名和形参中集合变量名相同，通过@RequestParam绑定参数关系
  • 日期：使用@DateTimeFormate
  • json参数：json数据键名和形参对象属性名相同，定义pojo类型形参接受，@RequestBody标识
  • 路径参数：通过请求url直接传递参数，使用{..}标识路径参数，使用@PathVariable获取路径参数

@RequestParam注解用于将请求参数绑定到方法的参数上。它可以用来指定参数的名称、是否必需、默认值

  • required：指定参数是否必需。默认为true。
  • defaultValue：指定参数的默认值。当请求中没有指定该参数时，将使用该默认值。

@ResponseBody 是一个注解，用于将方法的返回值直接作为 HTTP 响应的内容返回给客户端。在使用 Spring MVC 框架开发 Web 应用程序时，可以将 @ResponseBody 注解应用于方法级别或控制器级别，以指示方法的返回值应该被视为响应的主体内容。

使用 @ResponseBody 注解的方法会将返回值转化为合适的格式（如 JSON、XML 等），并将其作为 HTTP 响应的主体内容发送给客户端。这使得开发者可以方便地返回不同类型的数据，如对象、集合或纯文本，而无需显式地进行响应的构建。

在控制器类上添加 @RestController 注解，以将整个控制器类的所有方法的返回值都视为响应的主体内容。结合了 @Controller 和 @ResponseBody 注解的功能，用于直接返回数据对象而不进行视图解析。

IOC&DI

  • 控制反转 (IOC:inversion of cntrol):创建对象的控制权由程序自生转移到外部（容器：spring容器）
    • 注解@Component //将当前类交给IOC容器管理，成为bean对象
  • 依赖注入(DI:dependency injection)：容器为应用程序提供运行时，所依赖的资源
    • 注解@Autowired //IOC容器会提供该类型的bean对象，并赋值给该变量
  • Bean对象：在IOC容器中创建、管理的对象

Bean组件扫描

  • @SpringBootApplication具有包扫描功能，扫描当前包及其子包

  • 声明的四大注解如果要生效，需要被@ComponentScan(默认范围为<=当前包)扫描到,其他范围可以用过在启动项手动设置@ComponentScan

  • 建议将包放在启动项所在包内

Bean注入

@Autowired，遇到多个相同类型的bean时候，会出错

  • @Primary
  • @Autowired+@Qualifier(“bean名（类名小写）”)
  • @Resource(name=” bean名”)

@Resource和@Autowired 区别

@A是spring框架提供的注解，按照类型注入
@R是jdk提供的注解，按照名称注入

JDBC

JDBC(java database connectivity)是Java提供的标准数据库访问API，它定义了一套用于执行SQL语句和访问数据库的接口。使用JDBC，开发人员需要手动编写SQL语句、处理数据库连接、执行查询和更新操作等。JDBC提供了一些基本的接口和类，如Connection、Statement和ResultSet等，开发人员需要通过这些接口和类来完成数据库操作。

MyBatis是一个开源的持久层框架，它简化了Java与数据库之间的交互过程。MyBatis提供了一种将SQL语句与Java代码解耦的方式，通过XML配置文件或注解来定义和映射SQL语句。使用MyBatis，开发人员只需要编写SQL语句的映射，而不需要手动处理数据库连接和结果集等细节。MyBatis还提供了一些高级特性，如缓存机制、动态SQL、参数映射等，可以进一步简化数据库操作。
