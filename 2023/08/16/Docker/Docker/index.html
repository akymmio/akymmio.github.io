Docker

docker是一个用Go语言实现的开源项目，可以让我们方便的创建和使用容器，docker将程序以及程序所有的依赖都打包到docker container，这样你的程序可以在任何环境都会有一致的表现，这里程序运行的依赖也就是容器就好比集装箱，容器所处的操作系统环境就好比货船或港口，程序的表现只和集装箱有关系(容器)，和集装箱放在哪个货船或者哪个港口(操作系统)没有关系。

因此我们可以看到docker可以屏蔽环境差异，也就是说，只要你的程序打包到了docker中，那么无论运行在什么环境下程序的行为都是一致的，不会再有“在我的环境上可以运行”，真正实现“build once, run everywhere”。

此外docker的另一个好处就是快速部署，这是当前互联网公司最常见的一个应用场景，一个原因在于容器启动速度非常快，另一个原因在于只要确保一个容器中的程序正确运行，那么你就能确信无论在生产环境部署多少都能正确运行。

命令解析

docker run -d -p 80:80 --name nginx nginx

docker run 创建并运行一个容器，
-d 是让容器在后台一直运行
–name 为容器命名
-p 端口映射，格式为ip:hostPort:containerPort

-e 设置环境变量

还需要指定镜像名称，镜像名称可以指定镜像仓库中的镜像，也可以指定本地镜像 [repository]:[tag]

  • repository镜像名字
  • tag镜像版本

命令别名

在 ~/.bashrc 中添加

alias dps='docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Ports}}\t{{.Status}}\t{{.Names}}"'

生效：source ~/.bashrc

挂载

  • 数据卷挂载
  • 目录挂载

数据卷（volume）是一个虚拟目录，是容器内目录和宿主机之间映射的桥梁,方便操作容器内文件

挂载操作：在使用docker run 的时候
-v 数据卷:容器内目录
-v 本地目录:容器内目录 （以/开头否则识别为卷）

不指定的话会在宿主机上创建一个匿名卷（将数据挂载到宿主机上）

目录在官方镜像文档中查看

案例：修改nginx中的index.html文件

dockerfile

编写好了dockerfile，通过docker build 命令生成镜像
命令： docker build -t image dockerfileCategory

dockler build -t naginx .

-t:给镜像起别名repository:tag格式
. : 指定Dockerfile所在目录

build之后，创建并启动容器，然后就运行起来了

网络

ip地址因为经常变，所以要自定义网络，下次就不用ip来访问，而是通过容器名来访问


默认情况下，容器间是相互隔离的，要实现容器间通信，需要加入自定义网络
(可以通过创建容器的时候用 –network 连上,这时候没有默认网桥)

项目部署流程

  • 部署java项目，准备好jar包和Dockerfile
  • 构建镜像，docker build -t image .
  • 启动容器，docker run -d –name container -p 8080:8080 image
  • 部署静态资源，创建容器，挂载（容器要在同一个网络中）

DockerCompose


通过一个单独的docker-compose.yml文件，来定义一组容器服务，实现多个相互关联docker容器的快速部署
