Executor

Executor框架包括3大部分：

（1）任务。也就是工作单元，包括被执行任务需要实现的接口：Runnable接口或者Callable接口；

（2）任务的执行。也就是把任务分派给多个线程的执行机制，包括Executor接口及继承自Executor接口的ExecutorService接口。

（3）异步计算的结果。包括Future接口及实现了Future接口的FutureTask类。

Executor 接口定义了 execute()方法用来接收一个Runnable接口的对象，而 ExecutorService 接口中的 submit()方法可以接受Runnable和Callable接口的对象。

  • execute 方法只能提交任务，但无法获取任务执行的结果或者取消任务的执行；而 submit 方法可以提交任务，并且可以通过返回的 Future 对象来管理任务的执行。
  • submit 方法的返回值是 Future 对象，可以用于获取任务执行的结果，或者取消任务的执行；而 execute 方法没有返回值。

Future 是Java中的一个接口，位于 java.util.concurrent 包下，它代表了一个异步计算的结果。当你调用 ExecutorService 的 submit() 方法提交一个 Callable 或 Runnable 任务时，会返回一个 Future 对象。这个 Future 对象允许你在任务完成后检索结果、检查任务状态以及取消任务。

线程类型：

使用 Executors 工厂方法创建线程池简单方便，适用于一般的线程池需求；而直接使用 ThreadPoolExecutor 类则更适合对线程池进行定制和详细配置的情况。

反射

JAVA反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性；这种动态获取的信息以及动态调用对象的方法的功能称为java语言的反射机制。

两个应用场景

  • 和配置文件结合，动态的创建对象，并调用方法
  • 获取任意一个类的所有信息

Class对象：每当编写并编译了一个Java类后，都会有一个对应的 Class 对象在运行时生成，这个对象包含了关于该类的所有类型信息。

获取Class对象：

  • 编译阶段 Class.forName(“全类名”) 最为常用
  • 加载阶段 类名.class 当作参数进行传递
  • 运行阶段 对象.getClass() 已经有了类对象时

常用方法：

  • setAccessible()修改Java的访问限制
  • newInstance() 使用指定的Class对象创建新的实例

动态代理

动态代理是一种技术，可以在运行时动态地生成代理对象，而不是在编译时生成。它可以在不影响原始代码的情况下，增加额外的功能。

在Java中，动态代理主要使用Java反射机制实现。它允许开发者在运行时动态地创建一个实现一组给定接口的代理类，该代理类继承自java.lang.reflect.Proxy类，并且其实现方式是通过调用java.lang.reflect.InvocationHandler接口的invoke()方法来实现的。

动态代理
jdk代理

  • 定义一个接口及其实现类；自定义 InvocationHandler 并重写invoke方法，在 invoke 方法中我们会调用原生方法（被代理类的方法）并自定义一些处理逻辑；
  • 通过 Proxy.newProxyInstance(ClassLoader loader,Class<?>[] interfaces,InvocationHandler h) 方法创建代理对象；

cglib代理：允许我们在运行时对字节码进行修改和动态生成

  • 定义一个类；
  • 自定义 MethodInterceptor 并重写 intercept 方法，intercept 用于拦截增强被代理类的方法，和 JDK 动态代理中的 invoke 方法类似；
  • 通过 Enhancer 类的 create()创建代理类；
