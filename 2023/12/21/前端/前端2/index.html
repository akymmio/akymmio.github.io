VueRouter

路径和组件的映射关系
<router-link to=’ ‘>标签跳转
<router-view>显示组件

动态路由可选参数:在参数后面添加 ？

重定向：redirect:’/path’

404: path: ‘*’ (配置路由最后)

模式设置：

  • hash: 默认模式，带#
  • mode: ‘history’: 去掉#(需要服务支持)

编程式导航 (路由跳转):

路由传参：

  • path传参
  • name传参

都支持：

  • 动态路由: 接收：this.$route.params.xx
  • query传参: 接收：this.$route.query.xx

配置路由，例如：path: ‘/detail/:xx’,

路由返回：$router.back(‘ ‘),$router.go(-1)

组件缓存

路由跳转后组件被摧毁，返回来时组件被重建，所有数据要重加载

vuex

管理通用的数据，多组件共享的数据

在 Vuex store 的内部（如 mutations 和 actions）以及非 Vue 组件的 JavaScript 文件中，你可以直接通过 store 对象来访问和操作 Vuex store。这是因为这些地方可以直接访问 Vuex.Store 实例。

在 Vue 组件中，通常使用 $store 访问 Vuex store 的状态、getter、mutation 和 action

原生多了commit，辅助函数简化

vuex遵循单向数据流，组件中不能修改仓库中的数据
strict:true 开启严格模式,(上线时需要移除)

固定第一个state参数，其他是传递的形参

不可以传多个参数（可以封装为一个对象传入）

mutations：提供方法来修改state，组件中只需要提交对应的方法
mapMutations: 映射为组件中的方法,相当于提交，可以直接用mutations中的方法名


actions、mapActions和mutaions类似
