同步

async关键字用于声明一个函数为异步函数。这意味着该函数包含一些需要异步执行的代码，例如网络请求或定时器等。当一个函数被声明为async时，它内部的代码会以非阻塞的方式执行，而不会阻塞整个程序的执行。

await关键字用于等待一个异步操作的结果。它只能在async函数内部使用，并且用于等待一个Promise对象的结果。当await关键字被用于一个Promise对象时，它会暂停当前async函数的执行，等待Promise对象的结果。一旦Promise对象解析为成功状态，await表达式将返回Promise对象解析后的结果。如果Promise对象被拒绝，那么await表达式将抛出一个异常。

注意

导入问题：是否需要使用大括号 {} 取决于你所导入的内容是命名导出还是默认导出export default。对于命名导出（包括全部或部分导入），需要使用大括号；而对于默认导出，则不需要使用大括号。

当你在Vue中使用ref或Pinia状态管理时，如果你直接这样创建一个对store内部状态的引用：

const user = ref(userStore.user);
这里的user引用在创建时仅会得到userStore.user当时的值（在这个时刻，如果userStore.user还没有被填充数据，那么它可能就是undefined或初始值）。由于这个ref并不是响应式地链接到userStore.user的，所以后续userStore.user的更新不会自动反映到user上。

在 Vue 3 的 Composition API 中，ref 确实创建了一个响应式引用，但它只是响应式地管理该引用指向的单个值。当你使用 ref(userStore.user) 来创建一个响应式引用时，你实际上是在捕获 userStore.user 在那一刻的值，并将其封装在一个响应式的包装器中。但是，这个响应式引用并不会自动地与 userStore.user 保持同步，也就是说，如果 userStore.user 后续发生变化，user 引用本身不会自动更新。

这是因为 ref 只是创建了一个对值的引用，而不是对原始数据源的引用。ref 的响应性是基于它的值是否被重新赋值的，而不是基于它引用的原始数据是否发生变化。

在 Vue 的响应式系统中，ref 和 reactive 创建的响应式数据是 Vue 内部用来追踪依赖和触发更新的机制。但是，这种响应式机制是 Vue 内部的，它并不会自动监听 Vue 组件外部（如 userStore.user）的数据变化。
